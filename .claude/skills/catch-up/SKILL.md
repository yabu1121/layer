# データモデルドキュメントの同期

`docs/model/model.md` をコードの実際のデータモデルと一致させるための手順。

## 手順

1. 変更されたデータモデル定義（スキーマ、型、テーブル定義など）を特定する
2. `docs/model/model.md` の現在の記述を読む
3. 差分を洗い出す（追加されたフィールド / 削除 / 型変更 / リレーション変更）
4. `docs/model/model.md` を更新する。各エンティティについて以下を記載する:
   - エンティティ名と役割
   - フィールド一覧（名前 / 型 / 必須か / 説明）
   - 他エンティティとのリレーション
5. 変更履歴セクションに日付付きで1行追記する

## 出力フォーマット例

\`\`\`markdown
## User
ユーザーアカウント。

| フィールド | 型      | 必須 | 説明           |
|-----------|---------|------|----------------|
| id        | string  | ○    | 主キー         |
| email     | string  | ○    | ログインID     |
| createdAt | datetime| ○    | 作成日時       |

### 変更履歴
- 2026-05-19: email フィールドを追加
\`\`\`

## 注意
- ドキュメントとコードが食い違っている場合は、コード側を正としてユーザーに確認する
- 破壊的変更（フィールド削除・型変更）は変更履歴で目立つように記す
